# Domain Docs

This is a single-context repository.

Before engineering work, read:

- `CONTEXT.md` for canonical domain language.
- Relevant files under `docs/adr/` for binding architectural decisions.
- `DESIGN_DECISIONS.md` and `DESIGN.md` for product and visual contracts when the work touches the interface.

Use glossary terms exactly, including Structural Gap, Missing Transition, Relevance, Confidence, Suggested Event, Dismiss, and Test Stub. Surface any proposed change that contradicts an ADR instead of silently overriding it.
